什么是狭义HTML5？

  • 仍是HTML；
  • 新标记、新属性、旧标记和属性的增删改
  • 1999年制定的HTML 4.01和XHTML 1.0标准的升级版；
  • 前身是2004年WHATWG提出的Web Applications 1.0，2007年被W3C接纳；

早期的HTML语法被定义成较松散的规则，以助于不熟悉网络出版的人采用。但从2000起，web开发人员开始习惯使用XML语法的XHTML 1.0，虽然HTML5不是基于XHTML的，但建议使用XHTML的语法。这是一个良好的开始，也应该保持下去。一致的代码风格和规范，不仅爽心悦目，也让项目易于读写和维护。

本来计划用来替换HTML4.01的具有更严格语义和语法的XHTML 2.0，因为是一个彻底重构的激进版本，这一计划内部受阻后，HTML5才浮出水面。部分相关链接：Reinventing HTML

什么是广义HTML5？

  • 技术组合：包括HTML, CSS, JavaScript, Web Application API, 甚至SVG。类同ajax；
  • 现在热度无限HTML5，人们在谈起它时，更多是在阐述一种概念，这有点像当年的web2.0；
  • HTML5的出现，希望能减少对插件类的RIA的需求，如：flash, flex, silverlight等；

误区：一个普遍的误解是HTML5能够在网页中提供动画效果，这是不准确的。动画效果是由JavaScript和CSS实现的。

HTML5新特性

  • 新增更语义化和功能化标记，删除和修改部分旧标记；
  • 更简洁的代码；
  • CSS3
  • JS API；

HTML5标记变化

更语义化的新标记

<header>, <section>, <footer>, <nav>, <article>, <aside>等等

语义化的标记更能反映现代典型的web用法，将有利于搜索引擎的索引（即SEO）、小屏幕设备和视障人士使用，代码可读性增强；

更功能化的新标记

<audio>, <vedio>, <canvas>, <progress>, <meter>, <details>等等

  • vedio demo: Showcase of Apple, HTML5在线视频, videojs, OSM, Video For Everybody, FlareVideo

    video格式：Mpeg4(ch/sa/IE9), Ogg(ff/ch/op), WebM(ff/ch/op)

  • Audio Showcase of Apple HTML5
  • Progress Base Demo
  • Canvas Plasmatree、Canvas three、Canvas bomomo、Canvas mesmerizer、Canvas burn、Canvas tunneler

canvas 元素使用 JavaScript 在网页上绘制图像。canvas 元素本身是没有绘图能力的，所有的绘制工作在 JavaScript 内部完成。画布是一个矩形区域，您可以控制其每一像素。canvas 拥有多种绘制路径、矩形、圆形、字符以及添加图像的方法。

删除了部分旧标记

  • acronym, applet, basefont, big, center, dir, font, frame, frameset, isindex, noframes, strike, tt

这些被删除的旧标记要么可被其它标记代替，要么能用CSS实现，要么是无语义的

<acronym>之于<abbr>：

acronym与abbr标记都是用来表示缩写的，区别在abbr是常规的缩写，而acronym则是用来表示特定的首字母缩写。HTML5认为没有存在2个缩写标记的必要，于是选择移除acronym

<center>之于CSS align property

从前：你可能这样实现元素居中 
<center>这里是要居中显示的内容</center>
现在：我们更推荐这样实现 
div{text-align:center;}
                    <div>这里是要居中显示的内容</div>

修改了部分旧标记

  • a, b, cite, hr, i, input, menu, meta, s, small, u
这些被修改的标记，有的增加或删减了属性、功能，有的赋予了新的语义

a标记的修改之一：

从前：a只能包含一些inline级别元素 
<a href="#">inline node</a>
现在：a允许你包含一个段落 
<a href="#">
                    <p>A hyperlink that contains a paragraph.</p>
                    </a>
同时a标记的name属性被废弃，且新增了media属性，target属性也被修改

i标记被赋予的语义：

i标记现在被用来突出不同意见或语义

更简洁的代码

避免不必要的复杂

Doctype：

从前

<!DOCTYPE html PUBLIC "-//W3C/DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

现在

<!DOCTYPE html">

编码：

从前

<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />

现在

<meta charset="utf-8" />

同时，用link元素引用样式表，type="text/css"是可以省略的；script元素的type=”text/javascript”也可以不写

激动人心的JS API

Web Storage

  • sessionStorage
  • localStorage

客户端的本地存储能力，根据W3C Web Storage的说明，Web Storage的存储空间至少有5MB。Web Storage 分为两种， sessionStorage 和 localStorage，顾名思义，sessionStorage用于本地存储一个会话（session）中的数据，这些数据只有在同一个会话中的页面才能访问并且当会话结束后数据也随之销毁。因此sessionStorage不是一种持久化的本地存储，仅仅是会话级别的存储。localStorage用于持久化的本地存储，除非主动删除数据，否则数据是永远不会过期的。

Web Workers

允许客户端像C++等高级编程语言那样，将一些耗费时间的纯运算操作，以类似线程的方式来运行在非UI线程中，并以消息（Message）机制进行通信。Web Workers 是 HTML5 提供的一个javascript多线程解决方案，我们可以将一些大计算量的代码交由web Worker运行而不冻结用户界面。
